Die draw() Funktion wird in einer Endlosschleife immer wieder aufgerufen.

Normalerweise wird alles, was zu Beginn ausgeführt wird in der setup()-Methode
erstellt, alles was **immer wieder** passiert, in der **draw()**-Methode.

    p5.createCanvas(200, 200)

def draw():
    p5.circle(100, 100, 50)

p5.run()

Neue Konzepte:
* Variablen sind immer nur innerhalb der Funktion bekannt, in der sie definiert wurden.
* Möchte man auf eine Variable zugreifen, die außerhalb definiert wurde, so muss man einmal am Anfang der Funktion

Mit Variablen kann man noch mehr machen...

```python

python

python
import p5

show = True

def setup():
    p5.createCanvas(200, 200)

def draw():
    global show
    p5.clear()

    if show:
        p5.circle(100, 100, 50)

    show = not show

p5.run()
```

### Zufall

Experimentiere auch, wie du mit Hilfe des Zufalls Bilder zeichnen kannst:

```python

python

python
import p5

x = 100
y = 100

def setup():
    p5.createCanvas(200, 200)

def draw():
    global x, y
    p5.clear()
    p5.circle(x, y, 10)
    x += p5.random(-5, 5)
    y += p5.random(-5, 5)

p5.run()
```

### Mehrere Objekte

Du kannst auch mehrere Objekte animieren. Experimentiere mit folgendem Programm:

```python

python

python
import p5

xliste = [50, 100, 150]
frame = 0
speed = 1

def setup():
    p5.createCanvas(200, 200)

def draw():
    global frame, speed
    p5.clear()
    for x in xliste:
        p5.circle(x + frame * speed , 100, 20)
    frame = frame + 1

p5.run()
```
